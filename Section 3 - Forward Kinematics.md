<h1 id="section-3---forward-kinematics">Section 3 - Forward Kinematics</h1>
<h2 id="kinematic-chains-and-forward-kinematics">3.1 Kinematic Chains and Forward Kinematics</h2>
<h3 id="introduction">3.1.1 Introduction</h3>
<p><strong>Kinematic Chains:</strong> the general model of robot arms, fingers, legs,etc. contains two elements,</p>
<ul>
<li><strong>Links:</strong> rigid body</li>
<li><strong>Joints</strong>: two types, which are revolute joint (rotate around the axis) and prisimative joint (translate distance along axis).</li>
</ul>
<p>What is cared most is the location of <strong>end effector</strong> in space, then it becomes the analysis of <strong>forward kinematics</strong>.<br>
With the <strong>forward kinematics</strong> analysis, we can achieve the info about end effector by combining the info from every link of the robot, which also implies we can concern about the attitude of every link.</p>

<h3 id="urdf-full-notation-of-fk-in-computer">3.1.3 URDF: Full Notation of FK in computer</h3>
<p>The general ways to describe a robot are called <strong>Universal Robot Description Format (URDF)</strong>. There are many URDF, we are going to show one of these files in ROS.</p>
<pre><code>&lt;robot&gt;
	&lt;link name="L0"/&gt;
	&lt;joint name="J1" type="revolute"&gt;
		&lt;origin xyz="0 0 0.1" rpy="0 0 0"/&gt;
		&lt;axis xyz="0 0 1"/&gt;
		&lt;parent link="L0"/&gt;
		&lt;child link="L1"/&gt;
	&lt;/joint&gt;
	&lt;link name="L1"/&gt;
...
&lt;/robot&gt;
</code></pre>
<p>The above URDF is able to describe the robot, and more details can be read at <a href="http://wiki.ros.org/urdf">URDF in ROS</a>. You can see URDF can contain more than kinematics, such as shape info, inertial parameters, mass, collision, visual info, sensor info and so forth.</p>
<h3 id="summary">3.1.4 Summary</h3>
<p>Learning the kinematic chain and the file called URDF to store the kinematics in computers, we are already able to describe a robot arm.</p>
<p>However, URDF  is too general and takes too much memories to decribe the kinematics. As we can see, URDF is definately not good for the calculation of FK. Remember that the robot arms appeared in the age when the computers were not powerful as they are today, so we will introduce a classic way in next section, which is widely used for industrial robot arms.</p>
